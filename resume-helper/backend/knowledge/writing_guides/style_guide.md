# 简历写作风格指南

## Bullet结构规范

每条经历描述应遵循 STAR-K 结构：
- 动词开头（Action Verb）
- 行为对象（Task/Object）
- 方法/工具（Method/Tool）
- 可量化结果（Result）
- 关键词落点（Keyword）

示例：
- 弱：负责数据分析工作
- 强：使用Python和SQL搭建用户行为分析看板，覆盖5个核心业务指标，支撑产品迭代决策（月活提升12%）

## 量化指标指南

优先使用以下类型的量化指标：
- 规模类：处理X条数据、管理X个账号、覆盖X个城市
- 效率类：缩短X%时间、提升X%效率、节省X人力
- 增长类：增长X%、提升X个百分点、新增X用户
- 成本类：降低X%成本、节省X万元、ROI达到X
- 质量类：准确率X%、满意度X分、差错率降至X%

缺少具体数字时，使用[待补充]标记，不可编造。

## 语气风格

### 专业型（默认）
- 用词精准，避免口语化
- 强调方法论和工具栈
- 适合：技术岗、分析岗、管理岗

### 结果导向型
- 每条bullet以成果/影响开头
- 强调业务价值和ROI
- 适合：销售、运营、市场岗

### 学术型
- 强调研究方法和学术贡献
- 使用学术术语
- 适合：研究岗、高校、科研机构

## 中英文混排规范

- 中英文之间加空格：使用 Python 进行数据分析
- 专有名词保持英文：SQL、Python、Excel、Tableau
- 公司名保持原文：腾讯、Alibaba、ByteDance
- 学位缩写并列：工商管理硕士（MBA）

## 常见问题

### 避免的表述
- "负责"开头（改用具体动词：搭建、分析、推动、优化）
- "参与"（改为具体贡献：主导XX模块、负责XX环节）
- "协助"（改为：配合XX完成、支撑XX决策）
- 过于笼统的描述（"做了很多数据分析"→具体说明分析了什么、用什么方法、产出什么结果）

### 日期格式
- 统一使用：YYYY.MM - YYYY.MM
- 至今：YYYY.MM - 至今
- 不要混用不同格式
