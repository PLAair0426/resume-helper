# 隐私与数据治理策略

## 数据分类

| 级别 | 数据类型 | 处理规则 |
|------|---------|---------|
| L1-公开 | 模板、ATS规则、词库 | 无限制 |
| L2-内部 | 生成的简历版本、评分卡 | 用户可控TTL，默认30天 |
| L3-敏感 | 姓名、电话、邮箱、身份证号 | 加密存储，日志脱敏，不传第三方 |

## 数据保留策略（TTL）

- 上传的原始文件：处理完成后默认保留7天，用户可配置
- 结构化profile：默认保留30天
- 生成的简历版本：默认保留90天
- 操作日志：仅保留脱敏元数据，保留180天
- 用户可随时一键删除所有个人数据

## 第三方LLM调用策略

- 发送到LLM的内容默认脱敏（替换真实姓名/电话/邮箱为占位符）
- 用户可选择"完整发送"模式（需明确同意）
- 记录每次LLM调用的供应商、token数、是否脱敏

## 加密与访问控制

- 传输层：全站HTTPS/TLS
- 存储层：敏感字段AES-256加密
- 访问控制：RBAC（普通用户/管理员/审计员）
- API认证：JWT + Refresh Token
