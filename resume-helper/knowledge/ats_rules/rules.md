# ATS规则库

> 来源：Indeed、Greenhouse官方文档 + PDF方案总结

## 格式规范（自动检测项）

### 必须遵守
- 单栏布局（禁止多栏/分栏）
- 标准分区标题：教育经历 / 工作经历 / 项目经历 / 技能 / 证书
- 联系方式放在正文区域（非页眉/页脚/文本框）
- 日期格式统一（YYYY.MM 或 MMM YYYY）
- 文件大小 < 2.5MB（Greenhouse限制）
- 支持PDF和DOCX格式（DOCX通常更安全）

### 禁止使用
- 表格（会导致解析失败或部分失败）
- 页眉/页脚中放联系信息
- 文本框
- 图片/图形/图标
- 艺术字
- 分栏布局

### 关键词规范
- 缩写与全称并列（如 "MBA / Master of Business Administration"）
- 关键词自然融入 summary / skills / work experience
- 避免堆砌：每个关键词应有证据支撑（在哪个项目/职责中使用）
- 技能关键词应出现在技能表 + 经历bullet中（双重落点）

## 关键词权重体系

| 层级 | 权重 | 说明 | 示例 |
|------|------|------|------|
| Must-have | 5 | JD硬性要求 | "3年Python经验" |
| Title/Level | 4 | 职位名称/职级 | "高级数据分析师" |
| Core | 3 | 核心技能/职责 | "A/B测试、漏斗分析" |
| Evidence | 2 | 证据词 | "提升30%、处理10万条" |
| Nice-to-have | 1 | 加分项 | "有XX经验优先" |

## Bullet改写规则

### 结构
动词开头 + 行为对象 + 方法/工具 + 可量化结果/影响

### 示例
- 原句：负责活动数据分析
- 改写：使用SQL构建活动漏斗与留存看板，定位关键流失环节并提出优化建议（转化率提升X%）

### 禁止
- 不改变事实
- 缺失数字用 [待补充: 节省X%/提升Y%/处理Z条] 标记
- 不自动编造指标
